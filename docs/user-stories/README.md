# User Stories

Feature specifications grouped by role. Each role inherits capabilities from the roles below it.

```
employee → direct_manager → procurement_manager → admin
```

| Role                | Specification                                      |
| ------------------- | -------------------------------------------------- |
| Employee            | [employee.md](./employee.md)                       |
| Direct Manager      | [direct-manager.md](./direct-manager.md)           |
| Procurement Manager | [procurement-manager.md](./procurement-manager.md) |
| Admin               | [admin.md](./admin.md)                             |

Each document follows the same structure: **User story**, **Acceptance criteria**, **API Contract**, request examples, **Data model**, and **Error scenarios**.

For domain entities and business rules, see [equipment.md](../equipment.md).
